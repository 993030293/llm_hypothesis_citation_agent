# DeepScientist Integration

## Chosen Base Agent

This project chooses the official DeepScientist direction for Project A.

The official DeepScientist repository is cloned at:

```text
C:\Users\99303\git\DeepScientist-official
```

DeepScientist provides the scientific-research agent pattern:

```text
read papers -> find research gaps -> generate hypotheses -> prepare a research report
```

The added module in this repository is:

```text
Citation Evidence Chain Tracking Module
```

It is inserted after hypothesis generation and before final report writing.

## Why DeepScientist Matters Here

The course requirement is not just to build a standalone script. It asks for a
module added to an OpenClaw or DeepScientist research agent. In this project,
official DeepScientist is the base research agent. The new module checks whether
citations used to support DeepScientist-generated ideas are real, correctly
described, and actually supportive.

This makes the project focus clear:

```text
Official DeepScientist:
  generates research ideas and citation-backed claims

This added module:
  traces evidence chains and labels citations Green/Yellow/Red
```

## Module Boundary

The verification module receives citation-backed claims:

```json
{
  "claim_id": "C001",
  "hypothesis_id": "H001",
  "claim_text": "Prior work reports that ...",
  "cited_work": {
    "title": "...",
    "authors": ["..."],
    "year": 2024,
    "doi": "10.xxxx/xxxxx"
  }
}
```

It then performs:

```text
1. Existence check
   DOI/title lookup through public scholarly APIs.

2. Metadata check
   Compare title similarity, author overlap, year match, and DOI match.

3. Support check
   Compare the claim against public title/abstract/snippet evidence.

4. Color labeling
   Green: exists + metadata match + concrete support.
   Yellow: exists but support is partial or uncertain.
   Red: not found, metadata mismatch, or unsupported.
```

The labels are produced by code and public API evidence, not by an LLM decision.

## Adapter Entrypoint

The adapter is:

```text
agent/deepscientist_adapter.py
```

It can audit claims generated by the full workflow:

```powershell
python agent/deepscientist_adapter.py `
  --claims outputs/runs/20260530_152848/generated_claims.json
```

It creates:

```text
outputs/deepscientist_audits/<timestamp>/
  deepscientist_module_manifest.json
  input_claims.json
  source_hypothesis_payload.json
  citation_verification.csv
  evidence_chain.csv
  evidence_chain.md
  deepscientist_audit_summary.md
  tool_calls.jsonl
  evidence_items.jsonl
```

## Official DeepScientist Mapping

The official integration is the primary project route:

```text
Official DeepScientist quest
-> idea/write stage
-> citation_audit_claims.json
-> scripts/audit_deepscientist_output.py
-> citation_verification.csv + evidence_chain.csv
```

The local workflow can still run end to end as a legacy fallback for debugging,
but it is no longer the main demonstration route:

```text
PDF
-> pdf_reader
-> query_planner
-> literature_search
-> hypothesis_generator
-> deepscientist_adapter / citation_verifier
-> report_writer
```

The important interpretation for the presentation is:

```text
official DeepScientist + Claude Code:

deepscientist_adapter + evidence_chain_tracer + citation_verifier:
  are the added module for evidence-chain tracking and citation hallucination checking
```

## Path-2 Setup Scripts

Use these commands from the project root:

```powershell
python scripts/setup_path2_claude.py --force-skill
python scripts/path2_smoke_test.py
```

The first command prepares the official DeepScientist + Claude Code runtime and
installs the `citation-evidence-audit` skill. The second command creates a
small official DeepScientist quest, uses Claude Code to produce
`citation_audit_claims.json`, and audits it through this module.

## LLM Boundary

DeepSeek or another LLM may be added later for:

- more natural hypothesis wording;
- query expansion;
- final report summarization.

However, LLMs should not produce the final Green/Yellow/Red labels. The label
decision stays in the verifier code so the project does not use hallucination
to audit hallucination.
